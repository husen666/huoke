# 获客渠道中心 - 详细功能规格

## 1. 模块概述

获客渠道中心是火客系统的流量入口，负责统一管理所有外部获客渠道，实现线索的自动采集、清洗、评分和分配。

---

## 2. 渠道接入详细设计

### 2.1 企业微信（P0 - 最高优先级）

#### 接入方式
- **企业微信服务商 API**（推荐）：通过企业微信开放平台接入
- **企业微信自建应用**：企业自建应用模式

#### 获客能力
| 功能 | 实现方式 | 说明 |
|------|---------|------|
| 客户联系 | 企微「联系我」二维码 | 自动生成带参数二维码，客户扫码即加好友 |
| 渠道活码 | 一码多人轮询分配 | 多个销售轮流接待，负载均衡 |
| 群活码 | 群满自动建新群 | 社群裂变获客 |
| 欢迎语 | 好友添加自动发送 | 支持文字/图片/链接/小程序 |
| 客户标签 | 自动打标签 | 根据渠道来源自动标记 |
| 客户画像 | 聚合客户数据 | 聊天记录、行为轨迹统一汇总 |

#### 关键 API
```
POST /cgi-bin/externalcontact/add_contact_way     # 创建「联系我」
GET  /cgi-bin/externalcontact/get                  # 获取客户详情
GET  /cgi-bin/externalcontact/list                 # 获取客户列表
POST /cgi-bin/externalcontact/send_welcome_msg     # 发送欢迎语
POST /cgi-bin/externalcontact/mark_tag             # 打标签
回调事件：
  - 添加企业客户事件
  - 删除企业客户事件
  - 客户接替失败事件
```

#### 数据流
```
客户扫码 → 企微回调 → 系统接收 → 创建线索 → 自动标签 → 发送欢迎语 → 分配客服
```

## 3. 线索采集引擎

### 3.1 采集模式

#### 主动采集（Push）
- 各平台 Webhook 回调自动推送
- 表单提交自动创建
- 在线咨询自动记录

#### 被动采集（Pull）
- 定时任务轮询各平台评论/私信
- AI 扫描识别意向客户
- 竞品监控采集

### 3.2 线索处理管线 (Pipeline)

```
┌──────────┐   ┌───────────┐   ┌───────────┐   ┌───────────┐   ┌──────────┐
│ 原始数据  │──▶│ 数据清洗   │──▶│ 去重合并   │──▶│ AI 评分   │──▶│ 自动分配  │
│ 接收     │   │ & 标准化   │   │           │   │ & 分类    │   │          │
└──────────┘   └───────────┘   └───────────┘   └───────────┘   └──────────┘
```

#### Step 1: 数据清洗 & 标准化
- 手机号格式统一（+86、区号处理）
- 姓名规范化（去特殊字符）
- 地域信息标准化（省市区映射）
- 敏感信息过滤

#### Step 2: 智能去重
- **精确匹配**：手机号 / 微信号 / 邮箱完全一致
- **模糊匹配**：姓名 + 公司名相似度 > 80%
- **合并策略**：多渠道线索合并为一个客户，保留所有渠道记录

#### Step 3: AI 线索评分
```
输入 → DeepSeek 分析 → 评分 + 分类

Prompt 示例：
"根据以下客户信息进行意向评分(0-100)和分类：
 - 来源：抖音评论
 - 内容：'你们这个产品多少钱？有没有企业版？'
 - 用户画像：企业认证账号，粉丝2万
 
 输出格式：{score: 85, category: '高意向-价格咨询', reason: '...', suggested_action: '...'}"
```

#### Step 4: 自动分配规则

| 规则类型 | 说明 | 示例 |
|---------|------|------|
| 轮询分配 | 均匀分配给在线客服 | 线索依次分配给 A→B→C→A |
| 权重分配 | 按比例分配 | 高级销售 50%，初级销售 30%，新人 20% |
| 标签匹配 | 按客户标签匹配客服技能 | 技术咨询 → 技术客服，价格咨询 → 销售 |
| 地域分配 | 按客户地域分配 | 华东客户 → 华东团队 |
| VIP 优先 | 高分线索优先分配给 TOP 销售 | 评分>80 → 销售总监 |

## 5. 渠道 ROI 分析

### 5.1 核心指标

| 指标 | 计算公式 | 说明 |
|------|---------|------|
| 获客成本 (CAC) | 渠道花费 / 新增客户数 | 单个客户获取成本 |
| 线索成本 (CPL) | 渠道花费 / 线索数 | 单个线索获取成本 |
| 转化率 | 成交客户数 / 线索数 × 100% | 线索到成交转化 |
| 投资回报率 (ROI) | (收入 - 成本) / 成本 × 100% | 渠道投入产出比 |
| 客户终身价值 (LTV) | 平均客单价 × 复购次数 × 生命周期 | 客户长期价值 |
| LTV/CAC 比率 | LTV / CAC | > 3 为健康 |

### 5.2 分析维度
- **渠道维度**：按平台、按具体渠道（如抖音-直播 vs 抖音-评论）
- **时间维度**：日/周/月/季度趋势
- **活动维度**：按营销活动归因
- **内容维度**：按内容类型（短视频/图文/直播）

### 5.3 归因模型
- **首次触点归因**：归功于第一次接触的渠道
- **末次触点归因**：归功于成交前最后一次接触
- **线性归因**：平均分配给所有触点
- **时间衰减归因**：越接近成交权重越高
- **AI 智能归因**：DeepSeek 分析多触点路径自动归因
